# Data Storage

No traditional database. All data is stored as JSON files.

## Data Directory (data/)

| File | Description | Source |
|------|-------------|--------|
| `releases_raw.json` | Raw release data from GitHub/npm | `01_fetch_releases.py` |
| `changelog_parsed.json` | Parsed changelog entries | `02_fetch_changelog.py` |
| `npm_versions.json` | npm publish history | `03_fetch_npm.py` |
| `hn_signals.json` | Hacker News discussions about Claude Code | `04_fetch_hn.py` |
| `blog_posts.json` | Anthropic blog post metadata | `05_fetch_blog.py` |
| `enriched.json` | LLM-enriched data (summaries, importance) | `06_enrich_llm.py` |
| `auto_bundle.json` | Assembled intermediate bundle | `07_build_output.py` |
| `story_data.json` | Final storytelling data | `08_build_story_data.py` |
| `capabilities_seed.json` | Editorial seed: capability definitions | Manual / editorial |
| `epochs_seed.json` | Editorial seed: epoch definitions | Manual / editorial |

## Editorial Directory (data/editorial/)

Contains manually curated editorial content that feeds into the pipeline. These are the primary files to edit when updating narrative content.

## Web Output (web/)

| File | Description |
|------|-------------|
| `data.json` | Master bundle consumed by `prepare-data.ts` |
| `data.js` | JS module version of the same data |
| `index.html` | Legacy standalone HTML viewer |

## Generated Site Data (site/src/generated/)

| File | Description | Size |
|------|-------------|------|
| `story-home.json` | Homepage narrative data | ~153 KB |
| `explore-data.json` | Explorer release data | ~1.7 MB |

These are auto-generated by `bun run prepare:data` and should not be edited manually.

## Data Flow

```
External APIs (GitHub, npm, HN, Anthropic blog)
  --> scripts/01-05 (fetch)
  --> data/*.json (intermediate)
  --> scripts/06 (LLM enrichment)
  --> scripts/07-08 (assembly)
  --> web/data.json (master bundle)
  --> site/scripts/prepare-data.ts (Zod validation + transform)
  --> site/src/generated/*.json (site-ready)
```
